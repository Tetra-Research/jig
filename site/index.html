Show the move, not just the claim.

The right proof for jig is mechanical. A recipe, a small var payload, and a repeated edit applied the same way every time.

recipe.yaml 
ops:
  - inject: "{{ target_file }}"
    anchor:
      pattern: "^def {{ function_name | regex_escape }}\\("
    before: "^\\s+return \\{$"
    template: |
      logger.info(
          "{{ event_namespace }}.done",
          extra={
              "method": "{{ function_name }}",
              "step": "{{ step_name }}",
              "entity_id": {{ entity_id_expr }},
          },
      )
vars.json 
{
  "target_file": "services/core_service.py",
  "function_name": "create_record",
  "event_namespace": "core_service.create_record",
  "step_name": "validate_input",
  "entity_id_expr": "record_id"
}
before 
from datetime import datetime


def create_record(record_id):
    timestamp = datetime.utcnow().isoformat()
    return {
        "id": record_id,
        "created_at": timestamp,
    }
after 
from datetime import datetime
import logging

logger = logging.getLogger(__name__)


def create_record(record_id):
    timestamp = datetime.utcnow().isoformat()
    logger.info("core_service.create_record.done", ...)
    return {
        "id": record_id,
        "created_at": timestamp,
    }

Proof should match the claim.

The current harness compares a plain-language control skill against a jig-backed skill with the same target contract across five routine backend edit patterns. The question is not whether jig solves all coding. It is whether it helps on repetitive, shape-constrained work agents do constantly.

-1.46M tokens for jig across the three-rep run versus control.
-193s aggregate duration saved across the same head-to-head set.
-$4.06 aggregate cost delta in jig's favor on the replicated run.
15/15 pairs passed in both arms across the replicated head-to-head run.

Narrow claim, measured honestly.

For routine, pattern-heavy backend work, jig often cuts token usage, cost, and wall-clock time at equal correctness. It is not a claim about all coding tasks, and the site should not pretend otherwise.

Quick start in one loop.

Keep the first successful path short: install jig, run a recipe with vars, inspect the diff, and move the pattern into your agent workflow.

Run it

  1. Install the CLI.
  2. Pick an example recipe.
  3. Run jig with a concrete var payload.
  4. Inspect the resulting diff.
brew install jig
cp -R examples/structured-logging-contract/before demo
cd demo
jig run ../examples/structured-logging-contract/recipe.yaml \
  --vars "$(cat ../examples/structured-logging-contract/vars.json)"
