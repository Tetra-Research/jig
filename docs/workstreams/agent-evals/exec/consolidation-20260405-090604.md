## What Was Built

TypeScript evaluation harness under `eval/` that orchestrates LLM agent trials against fixture codebases, scores results on assertion pass rate / jig usage / file correctness, and generates aggregate reports. 6 scenarios across 4 tiers (easy, medium, discovery, error-recovery). 655-line test suite. All Django-style Python fixtures.

## Review Findings Summary

**Critical (4):** timeout scores not zeroed, scope extraction buggy, by_tier derived from tags not scenario field, by_category always empty.

**Major (9):** category not enforced, dry-run doesn't validate agent config, jig usage metrics incomplete (exit codes, valid_vars, within_expected_range), tokens/jig_calls not persisted, baseline comparison uses wrong metric, jig binary not on agent PATH, agents.yaml flags diverge from spec.

**Minor (3):** --print vs -p, historical results in report scope, test coverage gaps for critical behaviors.

Most findings are data completeness issues in scoring and reporting. The core architecture (sandbox isolation, trial loop, assertion scoring, JSONL persistence) is sound.
