## Open Decisions Requiring Sign-Off

### [HUMAN DECISION NEEDED] D-1: Render-all-upfront vs render-per-operation
Render ALL templates before ANY file write (fail-fast, no partial writes) vs render each template just before its operation. **Recommendation: upfront** per ARCHITECTURE D-1. Templates reference variables, not created files.

### [HUMAN DECISION NEEDED] D-2: Extra variables behavior
When user provides variables not declared in the recipe: **(a) pass through silently**, (b) warn, (c) error. **Recommendation: (a)** — LLMs construct variable objects with extra fields.

### [HUMAN DECISION NEEDED] D-3: Error accumulation vs fail-fast on operations
When op 2 of 5 fails: **(a) stop immediately**, (b) continue and report all. **Recommendation: (a)** — later inject may depend on prior create succeeding (AC-N6.2).
